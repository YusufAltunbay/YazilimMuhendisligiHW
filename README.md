# Mini-Dashboard Git Atölyesi

Bu proje, Git üzerinde takım çalışması, modüler kod geliştirme, `pull/push` döngüsü ve Merge Conflict (Çakışma) yönetimi pratikleri yapmak amacıyla hazırlanmıştır.

## Ekip Rolleri ve Dosya Dağılımı
* **Öğrenci A (Entegrasyon Lideri):** [Abdurrahman Ekin] - Hava durumu modülü (`weather.py`) ve uygulamanın son kontrolü.
* **Öğrenci B (Modül Geliştirici):** Yusuf Altunbay - Haber modülü (`news.py`) ve çakışma çözümü.

## Proje Dosyaları
* `main.py`: Uygulamanın ana dashboard dosyası.
* `weather.py`: Hava durumu bilgisini döndüren fonksiyon.
* `news.py`: Haber bilgisini döndüren fonksiyon.

## Teslimat Şartları: Süreçteki Deneyimlerimiz
Ödev yönergesi gereği, kasıtlı çakışma (conflict) yaratma ve çözme adımlarındaki hislerimiz aşağıdadır:

* **Öğrenci B :** Git'in eklediği işaretleri silip iki kodu birleştirdiğimde, Git'in aslında kodları ezmek yerine takım çalışmasını ne kadar güvenli hale getirdiğini hissettim.
* **Öğrenci A :** Çakışması çözülmüş ve birleştirilmiş kodları tek bir `git pull` ile sistemime sorunsuz dahil etmek, zaman makinesi gibi çalışan bu yapının pratikliğini hissettirdi.
